  <action>
    Create `scripts/sync-schedule.ts` following the same structure as `scripts/backfill.ts`.

    **Implementation steps:**

    1. Import `sql` from `./lib/db.js`, `fetchAll` from `./lib/bdl-client.js`, `upsertGames` from `./lib/upserts.js`.

    2. At startup, resolve LAC's BDL team_id from the teams table:
       ```typescript
       const [lacTeam] = await sql<{ nba_team_id: number }[]>`
         SELECT nba_team_id FROM teams WHERE abbreviation = 'LAC' LIMIT 1
       `;
       if (!lacTeam) throw new Error('LAC team not found in teams table — run npm run backfill first');
       const lacBdlTeamId = lacTeam.nba_team_id;
       ```

    3. Build date window: `startDate` = today minus 7 days, `endDate` = today plus 30 days. Format as `"YYYY-MM-DD"` strings using `new Date().toISOString().split('T')[0]` arithmetic.

    4. Fetch all Clippers games in the window from BDL `/games`:
       ```typescript
       const games = await fetchAll<BDLGame>('/games', {
         'team_ids[]': String(lacBdlTeamId),
         'dates[]': ...   // BDL supports multiple dates via repeated params
       });
       ```
       Note: BDL `/games` accepts `start_date` and `end_date` params (not `dates[]`). Use:
       ```typescript
       const games = await fetchAll<BDLGame>('/games', {
         'team_ids[]': String(lacBdlTeamId),
         start_date: startDate,
         end_date: endDate,
       });
       ```

    5. Determine the current season ID to pass to `upsertGames`. Use the year-based NBA season: if current month < 7 (July), seasonId = currentYear - 1, else seasonId = currentYear. (NBA season 2025-26 has seasonId 2025.)

    6. Call `upsertGames(games, seasonId)` — this handles ON CONFLICT upsert.

    7. Log progress:
       ```
       [sync-schedule] Fetching LAC games (2026-02-27 to 2026-04-05)...
       [sync-schedule] Found 12 games. Upserting...
       [sync-schedule] Done. 12 games synced.
       ```

    8. Handle `start_time_utc` gap: BDL game objects have a `status` field containing an ISO datetime string when the game is upcoming (e.g., `"2026-03-10T02:00:00.000Z"`). When mapping to the games table `start_time_utc`, check if `game.status` is a valid ISO datetime (contains "T") — if so, use it as `start_time_utc`. The `upsertGames` function in `upserts.ts` already handles this mapping; verify it maps `status` → `start_time_utc` when the status looks like a datetime. If it does not, add a note in the action log and extend the upsert mapping.

    9. End with `await sql.end()`.

    **Notes on upsertGames:**
    - `upsertGames` was designed for historical backfill — it already handles ON CONFLICT (nba_game_id) DO UPDATE
    - Check `scripts/lib/upserts.ts` upsertGames implementation to confirm it maps `home_team_score` / `visitor_team_score` → `home_score` / `away_score`; and `status` correctly. If the existing mapping needs to update `status` to 'in_progress'/'scheduled'/'final' from BDL's string values, add that conversion.
