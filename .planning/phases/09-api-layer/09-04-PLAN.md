    - Default active_only=true: filter by p.is_active=true unless ?active_only=false
    - Return players array sorted by display_name ASC
    - meta: source='db', ttl_seconds=3600

    - Try/catch wrapping; 500 on error
  </behavior>
  <action>
    Create src/app/api/players/route.ts:

    1. Parse query params: const url = new URL(request.url); const activeOnly = url.searchParams.get('active_only') !== 'false';

    2. SQL query:
    ```sql
    SELECT DISTINCT
      p.player_id::text,
      p.nba_player_id::text,
      p.display_name,
      p.position,
      p.is_active
    FROM players p
    WHERE EXISTS (
      SELECT 1 FROM player_team_stints pts
      JOIN teams t ON pts.team_id = t.team_id
      WHERE pts.player_id = p.player_id
        AND t.nba_team_id = ${LAC_NBA_TEAM_ID}
    )
    AND (${ activeOnly } = false OR p.is_active = true)
    ORDER BY p.display_name ASC
    ```

    NOTE: The conditional active_only filter — use two separate queries or sql template interpolation: if activeOnly is true, append AND p.is_active = true condition.

    3. Return { meta: buildMeta('db', 3600), players: rows }

<task type="auto" tdd="true">
  <name>Task 2: Implement GET /api/players/{player_id} detail endpoint</name>
  <files>src/app/api/players/[player_id]/route.ts</files>
  <behavior>
    - Parse player_id from route params (Next.js App Router: params.player_id as string)
    - If player not found: return 404 with buildError('NOT_FOUND', 'Player not found')
    - trend_summary: query rolling_player_stats WHERE player_id=id AND window_games=10 ORDER BY as_of_game_date DESC LIMIT 1; if no rows return null
    - charts.rolling_pts: query rolling_player_stats WHERE player_id=id ORDER BY as_of_game_date ASC; map to [{game_date, value: pts_avg}]
    - charts.rolling_ts: same but value: ts_pct
    - Both chart arrays empty [] when no rolling data — NEVER zeroes
    - splits: compute from game_player_box_scores JOIN games for home/away/wins/losses splits; null if no box score rows
    - game_log: query last 25 game_player_box_scores rows for player, JOIN games, JOIN teams for opponent abbreviation; format MIN from minutes column; null if no rows → return []
    - meta: source='mixed', ttl_seconds=600
    - Cache-Control: public, max-age=600
    - Try/catch, 500 on unexpected errors
  </behavior>
  <action>
    Create src/app/api/players/[player_id]/route.ts:

    Next.js App Router dynamic route signature:
    ```typescript
    export async function GET(
      request: Request,
      { params }: { params: { player_id: string } }
    ) { ... }
    ```

    1. Validate params.player_id is numeric string.

    2. Fetch player: `SELECT player_id::text, display_name, position FROM players WHERE player_id = ${params.player_id}::bigint`
       If no rows → return 404.

    3. Run parallel queries (Promise.all):
       - A: rolling_player_stats for trend_summary (latest window_games=10 row)
       - B: rolling_player_stats for charts (all rows ordered ASC)
       - C: game_player_box_scores for game log (last 25, JOIN games + teams)
       - D: splits computation — aggregate from game_player_box_scores JOIN games

    4. trend_summary: if query A empty → null. Else map first row to { window_games, minutes_avg, pts_avg, reb_avg, ast_avg, ts_pct, efg_pct } with ::float8 casts.

    5. charts: map query B rows ascending to rolling_pts and rolling_ts arrays.

    6. splits: compute home vs away, wins vs losses from game_player_box_scores.
       Home: WHERE g.home_team_id = player's team_id
       Away: WHERE g.away_team_id = player's team_id
       Win: WHERE (g.home_team_id = team_id AND g.home_score > g.away_score) OR (g.away_team_id = team_id AND g.away_score > g.home_score)
       For each split: AVG(points), AVG(ts_pct::float8)
       Return null if no game_player_box_scores rows at all.

    7. game_log: for each row, format FG as "${fg_made}-${fg_attempted}", 3PT as "${three_made}-${three_attempted}", FT as "${ft_made}-${ft_attempted}". home_away determined by whether game's home_team_id matches player's team_id. opp = opponent team abbreviation.

    8. Return full response with meta buildMeta('mixed', 600).
  </action>
