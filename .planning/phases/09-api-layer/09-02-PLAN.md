  <behavior>
    - NO_ACTIVE_GAME branch: live_snapshots table is empty OR no in_progress snapshot → return { meta(ttl=60), state:"NO_ACTIVE_GAME", game:null, key_metrics:[], box_score:null, insights:[], other_games:[], odds:null }
    - DATA_DELAYED branch: latest snapshot has payload.is_stale=true → return { meta(stale=true, stale_reason from payload, ttl=5), state:"DATA_DELAYED", ... data from stale snapshot }
    - LIVE branch: in_progress game snapshot found → compute key_metrics, call generateLiveInsights, call getLatestOdds, build full response
    - key_metrics computation from snapshot payload box score data (NOT from advanced_stats table):
      * eFG% = (FGM + 0.5 * FG3M) / FGA for LAC and OPP separately
      * TO margin = LAC_TOV - OPP_TOV
      * Reb margin = LAC_REB - OPP_REB
      * Pace = 48 * ((LAC_POSS + OPP_POSS) / 2) / game_minutes_played where POSS ≈ FGA - OREB + TOV + 0.44*FTA
    - key_metrics array has exactly 4 items in fixed order: efg_pct, tov_margin, reb_margin, pace
    - meta.stale for LIVE state comes from payload.is_stale, NOT recomputed
    - Cache-Control: no-store on all responses
    - Wrap entire handler in try/catch; return 500 with buildError on unexpected errors
    - Query games table to get nba_game_id, game_date, start_time_utc, home/away team info joined with teams table
    - box_score: build from payload.home_box and payload.away_box (BoxscoreTeam format from live snapshot payload)
    - other_games: query other in-progress games from live_snapshots (non-LAC) — return empty array if none
  </behavior>

    1. Imports: NextResponse from 'next/server', sql + LAC_NBA_TEAM_ID from src/lib/db.js, buildMeta + buildError from src/lib/api-utils.js, getLatestOdds from src/lib/odds.js, generateLiveInsights from src/lib/insights/live.js

    2. Query most recent live_snapshot:
    ```typescript
    const [snap] = await sql`
      SELECT
        snapshot_id,
        game_id::text AS game_id,
        period,
        clock,
        home_score,
        away_score,
        home_team_id::text AS home_team_id,
        away_team_id::text AS away_team_id,
        captured_at::text AS captured_at,
        payload
      FROM live_snapshots
      WHERE home_team_id = (SELECT team_id FROM teams WHERE nba_team_id = ${LAC_NBA_TEAM_ID})
         OR away_team_id = (SELECT team_id FROM teams WHERE nba_team_id = ${LAC_NBA_TEAM_ID})
      ORDER BY captured_at DESC
      LIMIT 1
    `;
    ```

    3. If no snap → return NO_ACTIVE_GAME response.

    4. Check snap.payload.is_stale → if true, return DATA_DELAYED response (still populate game data from the snapshot).

    5. For LIVE response:
       - Get game details from games table joined with teams WHERE game_id = snap.game_id
       - Compute key_metrics from snap.payload.home_box and snap.payload.away_box
         (determine which box is LAC using home_team_id match vs LAC internal team_id)
       - Build LiveSnapshot param for generateLiveInsights from snap fields
       - Call generateLiveInsights(liveSnap, { home_rolling_10: null, away_rolling_10: null })
         (rolling context can be null for now — live.ts handles null gracefully)
       - Map InsightCandidate[] to API response insight shape (proof: { summary: category, result: proof_result[0] })
       - Call getLatestOdds(snap.game_id)
       - Build box_score from payload.home_box/away_box player arrays
         (map BoxscorePlayer to API spec row: player_id, name, starter, MIN, PTS, REB, AST, STL, BLK, TO, FG "made-att", 3PT "made-att", FT "made-att", +/-)
       - Skip players with played === "0" (DNP)
       - other_games: empty array [] for now (acceptable for MVP — live data for non-LAC games is not stored)
       - Return full response with Cache-Control: no-store
