# Phase 9 Plan 02: GET /api/live Route Handler Summary

**GET /api/live endpoint delivering live Clippers game state — score, box score, on-the-fly eFG%/TO/reb/pace metrics, live insights, and odds from DB only (no CDN calls)**

## Performance

- **Duration:** 2 min

- **Completed:** 2026-03-07T01:08:37Z
- **Tasks:** 1
- **Files modified:** 1

## Accomplishments

- Implemented `GET /api/live` with 3 states: `NO_ACTIVE_GAME` (no snapshot), `DATA_DELAYED` (is_stale=true), and `LIVE` (active game)
- Key metrics computed on the fly from live snapshot box score payload — eFG%, TO margin, reb margin, pace — without querying advanced_stats table
- Full box score built from `payload.home_box`/`away_box`, filtering DNP players (played==="0")
- Live insights generated by calling `generateLiveInsights()` with null rolling context
- Odds fetched via `getLatestOdds()`, returned null if unavailable
- Cache-Control: no-store set on all responses; global try/catch with buildError for 500 errors

## Files Created/Modified

- `src/app/api/live/route.ts` — GET /api/live handler: 3-state game detection, key_metrics computation, box_score builder, insights/odds integration

## Decisions Made

- key_metrics computed entirely from `payload.home_box`/`payload.away_box` in `live_snapshots` — `advanced_stats` table is NOT queried (it only has post-game rows)
- `meta.stale` taken from `payload.is_stale` (not recomputed at request time) per plan spec
- `other_games` returns `[]` for MVP — non-LAC live snapshot data is not stored in the current schema
- `lacIsHome` determined by comparing `snap.home_team_id` to LAC's internal `team_id` fetched from `teams WHERE nba_team_id = LAC_NBA_TEAM_ID`
- Import paths use `../../../lib/*` (3 levels up from `src/app/api/live/`) matching existing route conventions established by schedule, history, etc.
