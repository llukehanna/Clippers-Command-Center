Create .github/workflows/post-game.yml for the nightly post-game pipeline.

Cron: `0 10 * * *` = 10:00 UTC = 2:00am PT (PST) / 3:00am PDT. Covers games ending by 1am PT.
Add `workflow_dispatch` for manual trigger.

The chain is: finalize-games → compute-stats → generate-insights.
If any step fails, GitHub Actions stops execution (default behavior with sequential steps in one job).

Use three sequential `run` steps in a single job (not separate jobs — this ensures step-level failure stops the chain without needing `needs:` dependency between jobs).

Each step uses the same env block with DATABASE_URL and BALLDONTLIE_API_KEY (finalize-games fetches from NBA CDN which uses the client in nba-live-client.ts — no key needed for CDN; compute-stats and generate-insights are pure DB operations, but include BALLDONTLIE_API_KEY for consistency):

```yaml
name: Post-Game Pipeline

on:
  schedule:
    - cron: '0 10 * * *'   # nightly at 2am PT
  workflow_dispatch:

jobs:
  post-game:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - name: Finalize games
        run: npm run finalize-games
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
          BALLDONTLIE_API_KEY: ${{ secrets.BALLDONTLIE_API_KEY }}
      - name: Compute stats
        run: npm run compute-stats
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
      - name: Generate insights
        run: npm run generate-insights
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
```

Note: Each step has only the env vars it needs. If finalize-games fails, compute-stats and generate-insights do not run (GitHub Actions default sequential behavior).
