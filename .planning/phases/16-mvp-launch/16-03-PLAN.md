<task type="auto">
  <name>Task 2: Vercel deployment</name>
  <files></files>
  <action>
Deploy the application to Vercel production.

Step 1 — Check if vercel CLI is available:
```bash
vercel --version
```
If not installed: `npm install -g vercel` or use `npx vercel`.

Step 2 — Deploy:
```bash
vercel --prod
```

If this is the first deploy, the CLI will prompt to link to a Vercel project. Link to the existing project or create a new one named "clippers-command-center".

Step 3 — Set environment variables in Vercel (if not already set):
```bash
vercel env add DATABASE_URL production
vercel env add BALLDONTLIE_API_KEY production
vercel env add ODDS_API_KEY production
```
Enter the values from .env.local when prompted. If vars are already set in Vercel dashboard, skip this.

Step 4 — Re-deploy after env vars are set (if they were just added):
```bash
vercel --prod
```

Capture the deployment URL (e.g., https://clippers-command-center.vercel.app).

Step 5 — Verify cron is active:
Visit Vercel Dashboard → Project → Cron Jobs tab. The poll-live cron should appear with schedule "* * * * *".
  </action>
  <verify>
    <automated>curl -s -o /dev/null -w "%{http_code}" "$(cat .vercel/project.json 2>/dev/null | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); try{const p=JSON.parse(d); console.log('https://'+p.projectId+'.vercel.app')}catch{console.log('URL_NOT_YET_KNOWN')}")" 2>/dev/null || echo "Run: curl -I https://your-deployment-url.vercel.app"</automated>
  </verify>
  <done>vercel --prod exits 0 and returns a deployment URL; app is accessible at the URL</done>
