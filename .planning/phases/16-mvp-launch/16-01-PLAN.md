  <name>Task 1: Stateless cron route handler</name>
  <files>app/api/cron/poll-live/route.ts</files>
  <action>
Create a Next.js App Router GET handler at app/api/cron/poll-live/route.ts that runs ONE poll cycle per invocation (no while-loop).

Logic flow:
1. Query games table for active/imminent Clippers game (same SQL as findActiveClippersGameInDB in poll-live.ts — copy inline).
2. If no active game: return NextResponse.json({ state: 'NO_ACTIVE_GAME' }, { status: 200 }).
3. Fetch scoreboard from NBA CDN (import fetchScoreboard, findClippersGame, LAC_TEAM_ID from scripts/lib/ via relative path ../../scripts/lib/... OR copy inline if path resolution fails).
4. If Clippers game not found in scoreboard: return { state: 'NO_ACTIVE_GAME' }.
5. Fetch boxscore and play-by-play with Promise.allSettled (non-fatal if either fails).
6. Build SnapshotPayload: { is_stale: false, stale_reason: null, home_box, away_box, recent_scoring }.
7. INSERT into live_snapshots (append-only, no ON CONFLICT).
8. UPDATE games table row (status, period, clock, scores).
9. If game.gameStatus === 3 (Final): call finalizeGame(gameDbId, nbaGameId) inline or import from scripts/lib/finalize.ts.
10. Return NextResponse.json({ state: 'OK', snapshot_written: true }, { status: 200 }).

Error handling:
- Wrap entire body in try/catch. On any error: console.error(err), return { state: 'ERROR', message: err.message } with status 200 (NOT 5xx — Vercel Cron retries on 5xx, which is not desired for a CDN outage).

Import strategy:
- Use `import { sql } from '@/src/lib/db'` for DB.
- For nba-live-client and poll-live-logic: try `import { fetchScoreboard, ... } from '../../../../scripts/lib/nba-live-client'` (relative path from app/api/cron/poll-live/). If TypeScript path resolution fails due to tsconfig rootDir, inline the minimal code instead.
- Inline the SnapshotPayload type (do not import from scripts/).
- Copy the extractRecentScoring and fetchRollingContext functions inline (they are small, pure, and avoid scripts/ boundary issues).

The cron route must respond within 60s (Vercel limit). Single NBA CDN fetch + DB write is well within this.

Do NOT add any authentication guard (e.g., CRON_SECRET header check) — this is a personal fan dashboard, not a public API.
