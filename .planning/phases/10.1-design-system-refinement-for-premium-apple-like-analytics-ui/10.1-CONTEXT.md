# Phase 10.1: Design System Refinement — Context

**Gathered:** 2026-03-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Refine the visual language of the existing UI shell (design tokens, typography, spacing, and all shared components built in Phase 10) to achieve a premium dark analytics aesthetic. No new features, no new pages, no data changes. Every decision here is about visual quality.

Deliverables: updated `globals.css` (tokens), refined TopNav, StatCard, BoxScoreTable, chart wrappers, skeletons, StaleBanner, and updated design-system MASTER.md to make the new direction canonical.

Phase 11 (Live Game Dashboard) builds on top of this — all page implementations inherit from what this phase establishes.

<decisions>
## Implementation Decisions

### Aesthetic direction
- **References (in priority order):** Apple Sports (hierarchy and restraint), Linear (spacing, borders, surfaces), Raycast (compact premium dark), Vercel dashboard (clean analytics), Superhuman (polish and typography)
- **Tone:** Calm, restrained. Prefer restraint over decoration at every decision point.
- **Not:** Generic admin dashboard feel, heavy cards, over-decorated surfaces, color overuse.
- **Density:** High information density but highly readable — Linear/Raycast compactness, not spacious.
- **Elevation model:** Subtle surface layering, not heavy raised cards. Surfaces should feel like they exist at different depths, not like floating boxes.

### Design tokens (globals.css)
- **Background:** Deep blue-black — darker and more neutral than current `#0F172A`. Move toward `#08090E` or `#090B10` range (near-black with a blue undertone, not pure black).
- **Elevated surfaces:** One distinct step up from background — subtle, not dramatic. Think `#0F1117` → `#141620` range. Cards and panels sit at this level.
- **Surface alt / active areas:** A second, slightly higher elevation level for selected/hover states.
- **Borders:** Hairline, very low opacity. `rgba(255,255,255,0.06)` style — visible but not assertive. No heavy `#334155` slate borders.
- **Primary text:** Near-white, high contrast — `#EDEFF2` or similar (not pure `#F8FAFC`, slightly warmer).
- **Secondary text / labels:** Dimmed, not just gray — `rgba(255,255,255,0.45)` or a cool `#6B7280` equivalent. Clearly subordinate.
- **Clippers red (`--primary`):** Keep `#C8102E` but use it sparingly — active states, live indicator, critical accents only. Not for decoration.
- **Clippers blue (`--secondary`):** `#1D428A` — used even more sparingly than red. Data highlights, not UI chrome.
- **Positive/negative:** Desaturated for premium feel. Green: `#34D399` (emerald-400-ish, softer than `#22C55E`). Red: `#F87171` (softer than `#EF4444`).
- **Chart palette:** Restrained 4-color set — muted blue, muted teal, warm amber, soft red. No saturated primaries. Designed to read on the dark background without visual aggression.
- **Highlight/amber:** Keep `#F59E0B` for live dot only. Not used elsewhere.

### Typography
- **Font:** Inter — keep. No change.
- **5-level scale (define as CSS custom properties):**
  - `--text-page-title`: Large, prominent — `1.5rem / 2rem`, `font-weight: 600`, tight tracking (`-0.02em`)
  - `--text-section-title`: Medium heading — `0.875rem / 1.25rem`, `font-weight: 600`, `letter-spacing: 0.02em`, ALL CAPS for section labels
  - `--text-stat-value`: Display number — `2rem / 2.5rem` or larger depending on context, `font-weight: 700`, `tabular-nums`, tight tracking (`-0.03em`). This is the dominant visual element in stat cards.
  - `--text-body`: Standard — `0.875rem / 1.5rem`, `font-weight: 400`
  - `--text-caption`: Small / metadata — `0.75rem / 1rem`, `font-weight: 400`, `secondary text color`
- **Stat numbers feel dominant:** StatCard's value should feel like the largest, clearest thing in its container. Label (section-title scale, ALL CAPS) sits above it, context (caption) sits below.
- **Table typography:** Headers use section-title scale (small, caps, muted). Cell values use body scale, right-aligned for numbers, tabular-nums.
- **Nav typography:** Labels at `0.8125rem` (13px), `font-weight: 500`, slight tracking. Wordmark at `0.875rem`, `font-weight: 700`.

### Spacing system
- **Base unit:** `4px`. All spacing is multiples of 4.
- **Named scale:**
  - `--space-1`: `4px` — icon gaps, micro spacing
  - `--space-2`: `8px` — inline, tight component internals
  - `--space-3`: `12px` — compact component padding
  - `--space-4`: `16px` — standard component padding
  - `--space-6`: `24px` — section gaps within a panel
  - `--space-8`: `32px` — between major sections
  - `--space-12`: `48px` — page-level vertical rhythm
- **Component defaults:**
  - Nav height: `48px` (down from `56px` / h-14) — tighter, more Linear-like
  - StatCard padding: `12px 16px` — compact, no excessive whitespace
  - Table row height: `36px` — dense but readable
  - Section gap (between stat card groups): `24px`
  - Page horizontal padding: `24px`
  - Dashboard grid gap: `16px`

### Navigation shell (TopNav)
- **Height:** `48px` (`h-12`) — down from `h-14`. Feels lighter, Raycast-like.
- **Background:** `bg-background/80 backdrop-blur-md` — more pronounced glass, slightly more transparent than current.
- **Bottom border:** `border-b border-white/[0.06]` — hairline, barely visible. Replaces current `border-border` (too heavy).
- **Wordmark:** Keep "CCC". Increase weight to `font-bold`, reduce size to `0.8125rem`, `tracking-widest`. More precise.
- **Nav links:** `0.8125rem`, `font-medium`. Active state: thin `1px` underline (use `border-b border-primary` but `border-[1px]` not `border-b-2`). Inactive: `rgba(255,255,255,0.45)`.
- **Link spacing:** Tighter — `gap-5` (20px) between links, `gap-6` between wordmark and links.
- **Live dot:** Keep amber pulsing dot — this is an important data-driven indicator, not decoration.
- **No right side content** — wordmark left, links immediate, clean.

### StatCard
- **Surface:** Use elevated surface token, NOT the card background. Aim for barely-visible lift over page background.
- **Border:** `border border-white/[0.06]` hairline — replaces current heavier border.
- **Padding:** `12px 16px` — compact.
- **Radius:** `8px` — slightly tighter than current `0.75rem`. More Linear, less Material.
- **Label (above value):** `--text-section-title` scale — `0.6875rem` (11px), `font-semibold`, `letter-spacing: 0.08em`, ALL CAPS, secondary text color.
- **Value:** `--text-stat-value` scale — `1.75rem` or `2rem`, `font-bold`, `tabular-nums`, primary text color. Visually dominant.
- **Delta/context (below):** `--text-caption` scale — `0.75rem`, secondary text color. Delta color: positive/negative tokens.
- **No hover lift** — stateful hover belongs on interactive elements. Stat cards are display-only.

### BoxScoreTable
- **Headers:** `0.6875rem`, `font-semibold`, `letter-spacing: 0.06em`, ALL CAPS, secondary text color. Aligned right for stat columns.
- **Cell values:** `0.8125rem`, `tabular-nums`, right-aligned for numbers.
- **Row height:** `36px` — compact.
- **Grid lines:** Remove vertical separators entirely. Horizontal: only very faint `border-b border-white/[0.04]` — nearly invisible. Rely on row height and spacing, not lines.
- **Hover state:** `bg-white/[0.03]` — extremely subtle, just enough to track the row.
- **Player name column:** Left-aligned, `font-medium`, primary text. Slightly more weight than stat columns.
- **Sticky header:** Keep — essential for usability.
- **No outer card border** — table floats with section context, not wrapped in a heavy card.

### Chart wrappers (LineChartWrapper, BarChartWrapper)
- **Chart palette:** Restrained 4 colors — define as tokens:
  - `--chart-1`: `#60A5FA` (blue-400, muted)
  - `--chart-2`: `#34D399` (emerald-400, muted)
  - `--chart-3`: `#FBBF24` (amber-400)
  - `--chart-4`: `#F87171` (red-400, muted)
- **Gridlines:** Horizontal only. `stroke-opacity: 0.08` — barely there. No vertical gridlines.
- **Axes:** Hidden axis lines (no `axisLine`). Tick labels at `0.6875rem`, secondary text color.
- **Tooltip:** Dark surface (`--surface-alt`), hairline border, `0.8125rem` text, clean value + label. No excessive padding.
- **Lines/bars:** Full opacity — the chart colors are already restrained, no need to dim further.
- **No chart titles** inside the wrapper — section titles above are sufficient. Wrappers are composable.
- **Minimal decoration:** No reference lines unless data-driven, no fill gradients under lines unless specifically called for.

### Skeletons
- **Base color:** `bg-white/[0.05]` — very dark, barely visible shimmer base.
- **Animation:** Subtle `animate-pulse` — keep current Tailwind pulse, don't over-animate.
- **Shapes:** Match the actual component dimensions exactly — skeleton must preview layout, not generic boxes.
- **StatCard skeleton:** Matches `12px 16px` padding, label height, value height, context height.

### StaleBanner (stale data indicator)
- **Placement:** Keep below nav, above content.
- **Style:** `bg-amber-950/40 border-b border-amber-800/30` — very subtle amber wash, not alarming.
- **Typography:** `0.75rem`, amber-400 text, icon + text. No bold, no all-caps.
- **Tone:** Informational, not urgent. "Data delayed — last updated X min ago."

### Live indicator (LiveDot)
- **Keep:** Amber pulsing dot as-is — it's already restrained and effective.
- **Size:** Keep `h-2 w-2` — small.
- **Colors:** Keep `bg-amber-500` / `animate-ping bg-amber-400` — this is intentional color for a meaningful signal.

### Design system file updates
- **Update `design-system/clippers-command-center/MASTER.md`** with the new canonical tokens and typography scale.
- **If any values conflict with the original MASTER.md** (original had light background, Fira fonts, blue primary): override them. The new direction is canonical; do not preserve conflicting rules for backward compatibility.
- **The MASTER.md becomes the source of truth** that planning agents use for future phases (11–14).

### Claude's Discretion
- Exact hex values within the ranges specified (e.g., choosing between `#08090E` and `#090B10` for background)
- Whether to use CSS custom properties or Tailwind `@theme` inline for the new spacing scale
- Whether chart wrappers accept a `palette` prop or default to the 4-token set
- Animation duration/easing for any transitions (should be `150ms ease` default)
- Exact responsive behavior within the existing 1024px minimum constraint

<specifics>
## Specific Ideas

- **Apple Sports reference:** Clean score hierarchy — large dominant number, small context label. Apply the same principle to StatCard: the number IS the content, everything else is support.
- **Linear reference:** Their table/list styling is the model for BoxScoreTable — ultra-fine borders, generous but not wasteful row height, excellent typographic contrast between header labels and data values.
- **Raycast reference:** Their command palette and compact UI — extremely tight padding, restrained color use, strong typographic hierarchy does the work that color and decoration usually do.
- **Vercel dashboard:** Clean analytics grid — flat sections with clear visual grouping, no card outlines on everything, whitespace as separator.
- **Superhuman:** Typography precision — every font-size, weight, and tracking value is intentional. Labels, values, and captions are clearly distinct at a glance.
- **"Avoid over-carding":** Not every section needs a card border. Layout regions can be separated by spacing and subtle headers, not by drawing a box around everything.

</specifics>
