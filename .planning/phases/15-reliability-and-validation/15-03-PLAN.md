Purpose: Closes RELY-03 — the UI must remain functional and meaningful during upstream outages, with accurate "last updated" timing.

Output: Fixed API + banner, human visual verification of the DATA_DELAYED state.

The `snap.captured_at` (when last data arrived) is NOT in this response — only in meta.generated_at (which is now()).

Fix plan:
1. In `app/api/live/route.ts`: Add `snapshot_captured_at: snap.captured_at` to the DATA_DELAYED response body (not in meta — add as a top-level field).
2. In `components/stale-banner/StaleBanner.tsx`: Add `capturedAt?: string` prop. Prefer `capturedAt` over `generatedAt` for the minutes-ago calculation. Keep `generatedAt` prop for backward compat (other usages, if any).
3. In `app/live/page.tsx`: Pass `capturedAt={data?.snapshot_captured_at}` to StaleBanner.
