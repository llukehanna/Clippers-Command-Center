    }

    Rendering:
    - When metrics is null/undefined: render a flex row of 5 StatCardSkeleton (equal width, gap-3).
    - When metrics is loaded: render a flex row of 5 StatCard components.
      - First 4 cards: map over API metrics[] array in order (efg_pct, tov_margin, reb_margin, pace)
      - 5th card: FT Edge — computed via computeFtEdge(lacFt ?? '0-0', oppFt ?? '0-0')
        label="FT Edge", value formatted as string with sign (e.g., "+6" or "−4" or "0")
        positive={ftEdge > 0}, delta={ftEdge !== 0 ? (ftEdge > 0 ? '+' : '') + ftEdge : undefined}

    Value formatting per metric key:
    - 'efg_pct': `${(value * 100).toFixed(1)}%` — delta formatted same way with sign
    - 'tov_margin': format as integer with sign: `${delta_vs_opp > 0 ? '+' : ''}${Math.round(value)}`
    - 'reb_margin': same as tov_margin
    - 'pace': `${value.toFixed(1)}` — pace is team's own value, not a margin; delta shows vs opp
    - delta_vs_opp: show as signed string when not null; positive when > 0

    Layout: `flex flex-wrap gap-3` — allows wrapping on narrow viewports instead of horizontal scroll
    (per Claude's Discretion — wrapping chosen over scroll).
    Each StatCard: `flex-1 min-w-[100px]` so cards share width equally but can wrap.
