<objective>
Fix the case mismatch between game status values written by BDL sync ('Final') and queried by finalize-games/API routes ('final'), then re-run the data pipeline to populate historical games and player stats.

Purpose: The root cause of both BUG-04 and BUG-05 is that `normalizeBDLStatus` in upserts.ts maps BDL's `"Final"` to `status: 'Final'` (capital F), but `finalize-games.ts` queries `WHERE g.status = 'final'` (lowercase). This means finalize-games finds 0 games to process, leaving game_player_box_scores and rolling_player_stats empty. The API routes for schedule and home also use lowercase `'final'`, compounding the issue.

Output: Consistent lowercase `'final'` status across the entire codebase and database. After pipeline re-run, historical and player data fully populated.
</objective>
