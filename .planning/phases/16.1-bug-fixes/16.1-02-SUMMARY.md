## What Was Built

The DATA_DELAYED branch in `app/api/live/route.ts` was returning `box_score: null` and `key_metrics: []` even though the stale snapshot payload contains full box score data (home_box/away_box). This caused users to see empty skeletons with no data when the poll daemon was offline.

The fix adds, after `fetchGameDetails`, the same LAC team lookup pattern used in the LIVE branch:
1. Query `teams` table for LAC's internal `team_id` and `abbreviation`
2. Determine if LAC is home or away from `snap.home_team_id`
3. Extract `lacBoxStale` and `oppBoxStale` from `payload.home_box`/`payload.away_box`
4. Call `computeKeyMetrics(lacBoxStale, oppBoxStale)` — replaces `key_metrics: []`
5. Call `buildBoxScore(lacBoxStale, oppBoxStale, lacAbbrStale, oppAbbrStale)` — replaces `box_score: null`
6. Fetch odds via `getLatestOdds(snap.game_id)` — replaces `odds: null`
